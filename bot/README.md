# Telegram-бот @vkr_reviews_analyzer_bot

Бот к LLM-конвейеру из основной системы. Может работать как демо (показывать результаты эксперимента), как универсальный анализатор текстов/файлов в личке и как сборщик-анализатор групповых чатов.

## Что умеет

### Просто пишите боту

| Что отправить | Что получите |
|---|---|
| **Любой текст** в личке | LLM сделает аналитический разбор: тональность, темы, проблемы, рекомендации |
| **CSV / XLSX / TXT / JSON файл** (до 20 МБ) | Пакетный анализ содержимого — бот сам находит текстовую колонку (text / review / comment / message / body / content) и анализирует первые 20 фрагментов |

### Команды в личке

| Команда | Что делает |
|---|---|
| `/start`, `/help` | Описание и список команд |
| `/analyze` | Живой анализ 5 случайных отзывов из RuReviews-500 (~30 сек) |
| `/report` | Готовый аналитический отчёт по 20 отзывам |
| `/stats` | Статистика тестовой выборки (500 отзывов) |
| `/alerts` | Топ негативных тем с алертами |
| `/compare` | Сравнение метрик 6 моделей |
| `/chats` | Список подключённых групповых чатов — выберите кнопкой |
| `/chat_stats` | Статистика выбранного через /chats чата |
| `/chat_analyze` | Анализ выбранного чата с выбором периода (1 день / 3 дня / неделя / месяц / свои даты) |

### В группе (нужны права админа)

| Команда | Что делает |
|---|---|
| `/chat_stats` | Статистика этого чата |
| `/chat_analyze` | Анализ этого чата → выбор периода → LLM-отчёт |

## Как запустить локально

### 1. Поставить Ollama и модель

Скачать установщик: https://ollama.com/download

```bash
ollama pull qwen2.5:7b
```

Ollama стартует автоматически как служба на http://localhost:11434.

### 2. Установить зависимости

```bash
pip install -r ../requirements.txt
```

### 3. Получить токен бота

У [@BotFather](https://t.me/BotFather) в Telegram: `/newbot` → следуйте инструкциям → получите токен вида `1234567890:AAxxxxxxxxxxxxxxxxxxxxxx`.

### 4. Запустить

```bash
# Linux / Mac
export BOT_TOKEN="ваш_токен"
python telegram_bot.py

# Windows PowerShell
$env:BOT_TOKEN="ваш_токен"
python telegram_bot.py
```

После запуска бот начнёт polling и будет ждать сообщений.

## Как добавить в групповой чат

1. Добавьте бота в чат (через поиск по юзернейму)
2. Сделайте его **администратором** — иначе Telegram не отдаёт сообщения боту
3. Напишите в чате несколько сообщений
4. В личке с ботом наберите `/chats` — увидите чат в списке
5. Выберите чат → `/chat_stats` или `/chat_analyze`

Один бот может работать с любым количеством чатов параллельно. Каждый чат хранится отдельно в памяти (до 200 последних сообщений на чат).

## Архитектура

```
Пользователь → Telegram API → aiogram 3 → telegram_bot.py
                                              ├── Ollama (localhost:11434)
                                              │     └── qwen2.5:7b
                                              └── research/data + results
```

Бот переиспользует данные из соседней папки `research/` — пути ищутся автоматически (`../research/data/test_set_500.csv`, `../research/results/*.json` и т. д.).

## Что использует под капотом

- **aiogram 3** — асинхронный фреймворк для Bot API
- **Ollama** — self-hosted LLM (по умолчанию `qwen2.5:7b`)
- **pandas + openpyxl** — парсинг CSV / Excel из загруженных файлов
- **requests** — для вызова Ollama API

## Зачем self-hosted

Для корпоративного развёртывания **критично, чтобы данные не уходили в OpenAI / Anthropic** (ФЗ-152 «О персональных данных»). Ollama + Qwen 2.5 решает: модель крутится локально, ничего не утекает за периметр.

В исходных контрагентских отзывах часто есть персональные данные (имя, адрес доставки, номер заказа) — поэтому облачные LLM сразу за бортом.

## Лимиты

- Файл до 20 МБ (ограничение Telegram Bot API)
- Анализируются первые 20 фрагментов файла (чтобы не перегружать LLM)
- В групповом чате хранится 200 последних сообщений на чат (in-memory)
- При перезапуске бота буфер чатов сбрасывается (для production нужно подключить БД)
